Github - principais comandos para trabalhar Github

Configuração do projeto na maquina local

  1. Criar um repositório no Github
  2. Clonar o prjeto no PC

    git clone "nome_do_repositório_criado"

  3. Configuração usuario

    git config user.name "seu_nome"

  4. Configuração email

    git config user.email "nome@email.com"

Comandos básicos

Os proximos comando basicos 'checkout -b'², 'adicionar'⁴, 'commit'⁵, 'push'⁶, 'pull'⁸.

  1. Verificar ramo atual

    git branch

  2. Criando um novo ramo e entrar

    git checkout -b "nomebranch"

  3. Momento para adicionar arquivos e codigos do seu projeto referente a demanda atual da tarefa.

  4. Adicionar alterações para comitar

    git add .

  5. Commitar codigo

    git commit -m"feat: resumo da SUA atividade"

  6. Enviar codigo para o git

    git push origin "nomebranch"

  7. Abrir a PR para realizar o merge da branch com seu novo código com a main.

    • Após o comando "git push", aparece o link no terminal

    • Direciona para site do Github comparando diferença do codigo com a branch principal

    • Unir o código com a branch principal no botão "Merge pull request"

  8. retornar para a main

    git checkout main

  9. atualizar branch main

    git pull origin main
